## File Processing Pipeline

### 11-Step Validation Pipeline

```
1. File Detection (chokidar)
   в””в”ҖвҶ’ File added or changed event

2. Validate Existence
   в””в”ҖвҶ’ Check file exists and is readable

3. Validate Extension
   в””в”ҖвҶ’ Only .epub and .pdf allowed

4. Skip Special Files
   в””в”ҖвҶ’ Ignore .icloud, dotfiles, temp files

5. Validate Size
   в””в”ҖвҶ’ Reject files > 100MB (configurable)

6. Validate Age
   в””в”ҖвҶ’ Skip files < 5 seconds old (still syncing)

7. Calculate Hash
   в””в”ҖвҶ’ SHA-256 hash for deduplication

8. Check Local State
   в””в”ҖвҶ’ Is file already processed?

9. Check Backend
   в””в”ҖвҶ’ Does hash already exist on backend?

10. Upload File
    в””в”ҖвҶ’ POST multipart/form-data with Bearer token

11. Mark Processed
    в””в”ҖвҶ’ Save to state, mark complete
```

## Key Features

### 1. Smart Deduplication
- SHA-256 hashing matches backend algorithm
- Checks local state first (fast)
- Falls back to backend check (accurate)
- Prevents redundant uploads

### 2. Reliable Retry Logic
- Exponential backoff: `baseDelay * 2^(retry-1)`
- Jitter: Вұ10% of delay
- Configurable max retries (default 3)
- Example timing:
  - Attempt 1: Immediate
  - Attempt 2: ~1s
  - Attempt 3: ~2s
  - Attempt 4: ~4s

### 3. Concurrency Control
- Configurable concurrent uploads (default 2)
- Prevents parallel uploads of same file
- Queue-based processing
- Respects rate limits

### 4. iCloud Sync Awareness
- Skips .icloud placeholder files
- Waits for minimum file age (prevents partial uploads)
- Works with any iCloud Drive path

### 5. Persistent State
- JSON-based state file
- Tracks processed files with metadata
- Tracks failed files with retry counts
- Survives agent restarts

### 6. Graceful Shutdown
- Signal handlers (SIGINT, SIGTERM)
- Queue draining before exit
- State persistence on shutdown
- Clean exit logging

## API Integration

### Upload Endpoint
```http
POST /api/ebooks/upload HTTP/1.1
Content-Type: multipart/form-data
Authorization: Bearer {agentToken}

file: {binary content}
sourceType: agent
```

### Response Codes
- **201 Created** - Success, returns {id, title}
- **409 Conflict** - Duplicate hash
- **401 Unauthorized** - Invalid token
- **5xx** - Server error (will retry)

## Configuration

### Environment Variables

**Required:**
- `ICLOUD_WATCH_PATH` - iCloud folder to watch
- `BACKEND_URL` - Backend API URL
- `AGENT_TOKEN` - Bearer token for auth

**Optional (with defaults):**
- `LOG_LEVEL` - debug, info, warn, error (default: info)
- `MAX_FILE_SIZE_MB` - File size limit (default: 100)
- `MIN_FILE_AGE_SECONDS` - Min file age (default: 5)
- `MAX_RETRIES` - Retry attempts (default: 3)
- `RETRY_DELAY_MS` - Base retry delay (default: 1000)
- `QUEUE_CONCURRENCY` - Parallel uploads (default: 2)
- `POLLING_INTERVAL_MS` - Watcher poll interval (default: 2000)
- `STATE_FILE` - State file path (default: ./agent-state.json)

## Troubleshooting

### Common Issues

**Files not uploading:**
- Enable debug logging: `LOG_LEVEL=debug npm start`
- Verify paths exist and are readable
- Check file extension (.epub or .pdf)
- Ensure file size < 100MB
- Wait 5+ seconds (file age requirement)

**Backend connection fails:**
- Verify `BACKEND_URL` is correct
- Check `AGENT_TOKEN` is valid
- Ensure backend is running
- Check network connectivity

**High memory usage:**
- Reduce `QUEUE_CONCURRENCY`
- Check for hung uploads in queue
- Monitor `agent-state.json` size
